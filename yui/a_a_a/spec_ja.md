# Yui 言語 仕様書（日本語）

Yui は、日本語による簡単な教育用プログラミング言語です。
プログラムを構成する最小限の要素（型・式・文）のみを備え、
小中学生でも自然な日本語で読み書きできることを目指しています。

---

## 1. 字句

ソースコードを構成する最小単位（トークン）の規則です。

| 要素 | 例 |
|------|----|
| 空白 | 半角スペース, タブ, 全角スペース `　` |
| 改行 | `\n`（文の区切り） |
| 行コメント | `#` または `＃` から行末まで |
| 識別子 | `[A-Za-z_][A-Za-z0-9_]*`、および `「…」` で囲む拡張名 |

文は改行で区切られる。1 行に 1 文を書くのが基本です。

識別子には日本語も使えます（例: `絶対値`, `最大値`, `標準ライブラリを使う`）。

---

## 2. 型システム

Yui の値はすべて以下の 7 種類いずれかの型を持ちます。

| 型 |  リテラル |
|----|----------|
| ヌル |  `値なし` |
| 論理値 |  `真` / `偽` |
| 整数 |  `42` |
| 小数 |  `3.14`（表示時は小数点以下 6 桁固定） |
| 文字列 |  `"あ"`, `"x={x}"` |
| 配列 |  `[1, 2, 3]` |
| オブジェクト |  `{"x": 1, "y": 2}` |

- 文字列は、内部的に**文字コード（int）の配列**として保持される。これにより文字列と配列は同じ操作（要素追加・長さ取得など）で扱える。
- 整数も同様に、内部的には**2 進数のビット配列（LSB が先頭）**として保持される。したがって整数にもインデックス・長さ取得・要素追加などの配列操作が暗黙に適用できる（例: `6` は `[0,1,1]` として扱え、`6[0]` は `0`、`6[1]` は `1`、`6の大きさ` は `3`）。この性質を使えば、ビット演算（NOT・AND・OR・XOR・シフト・加減算・2 の補数）を純粋な配列操作として実装できる。
- 配列等値比較は再帰的で、文字コード配列と文字列、ビット配列と整数は相互比較可能（例: `[72,105] == "Hi"` は真、`[0,1,1] == 6` は真）。

---

## 3. 式

値を生み出す構文要素です。式は他の式の中に入れ子にでき、最終的に 1 つの値へ評価されます。

### 3.1 リテラル

ソース中に直接書かれる値の表記です。

| 型 |  リテラル |
|----|----------|
| ヌル |  `値なし` |
| 論理値 |  `真` / `偽` |
| 整数 |  `42` |
| 小数 |  `3.14`（表示時は小数点以下 6 桁固定） |
| 文字列 |  `"あ"`, `"x={x}"` |
| 配列 |  `[1, 2, 3]` |
| オブジェクト |  `{"x": 1, "y": 2}` |

### 3.2 文字列補間

文字列の中に式を埋め込み、結果を文字列化して連結します。

`"{expr}"` で式を埋め込み可能。`{` `"` 自体を文字として入れたいときは `\{` `\"` とエスケープします（`\\`, `\n`, `\t` も使えます）。

```
name="ゆい"
msg="こんにちは、{name}さん"
```

### 3.3 長さ演算

配列・文字列(expr)の要素数を取得します。

`exprの大きさ`

### 3.4 単項演算

ひとつの式に作用する演算子です。

`-expr`（符号反転）。リテラルに直接付けて `y=-2` のように書けます。

### 3.5 二項演算

Yui では算術記号を使わず、関数呼び出しの形で書きます。

**重要**: yui は `+ - * / %` は**無効**。標準ライブラリの関数（`和`, `差`, `積`, `商`, `剰余`）を使う。

| 関数 | 意味 |
|------|------|
| `和(x,…)` | 合計（引数が配列なら展開） |
| `差(x,y,…)` | 差 |
| `積(x,y,…)` | 積 |
| `商(x,y)` | 商（int/int は床除算） |
| `剰余(x,y)` | 剰余 |

### 3.6 インデックス / プロパティ

配列やオブジェクトから要素を取り出します。取り出した場所に代入もできます。

- `a[i]` → 配列 a の i 番目の要素（0 から始まる）
- `O["x"]` → オブジェクト O の `"x"` プロパティ。キーは文字列を使う
- `aの大きさ` → 配列 a（または文字列 a）の要素数

```
A=[10,20,30]
A[1]=200         # 2 番目の要素を書き換え

O={"x":0,"y":0}
O["x"]=1         # プロパティに代入
```

### 3.7 関数適用

定義済みの関数を呼び出して値を得ます。

`f(a, b, c)` — 引数は `,` 区切り。

## 4. 文

プログラムの実行単位です。文は上から順に評価され、変数の更新や制御フローを行います。

### 4.1 代入

変数に値を結びつけます。同じ名前に再代入すると上書きされます。

```
x = 式
```

### 4.2 増減

数値変数を 1 だけ増やす／減らすための簡略構文です。

```
xを増やす
xを減らす
```

### 4.3 追加

配列の末尾に値を 1 つ加えます。

```
aにxを追加する
```

文字列も文字コードの配列なので追加できる。

### 4.4 条件分岐

条件が成り立つかどうかで実行する処理を切り替えます。

```
もしxがyならば {
   …
}
そうでなければ {
   …
}
```

そうでなければ省略可能です。

#### 比較の書き方（接尾辞形式）

`もし <式1> が <式2> <接尾辞> ならば` の形で、接尾辞によって比較の種類が変わります。

| 接尾辞 | 意味 | 例 |
|--------|------|----|
| （なし） | `==`（等しい） | `もしxが0ならば` |
| `以外` | `!=`（等しくない） | `もしxが0以外ならば` |
| `より小さい` | `<` | `もしrがbより小さいならば` |
| `より大きい` | `>` | `もしxが10より大きいならば` |
| `以下` | `<=` | `もしdistが1以下ならば` |
| `以上` | `>=` | `もしnが100以上ならば` |
| `のいずれか` | `in`（含まれる） | `もし"banana"がfruitsのいずれかならば` |
| `のいずれでもない` | `not in`（含まれない） | `もし"grape"がfruitsのいずれでもないならば` |

`==` `!=` `<` `<=` `>` `>=` などの**記号による比較は yui 構文では使えません**。必ず `が〜ならば` と接尾辞の組み合わせで書きます。

#### 多段分岐（elif 相当）

Yui には `elif` 専用のキーワードはありません。`そうでなければ` の中にさらに `もし` をネストして表現します。

```
もしfizzが0ならば {
   もしbuzzが0ならば {
      resultに"FizzBuzz"を追加する
   }
   そうでなければ {
      resultに"Fizz"を追加する
   }
}
そうでなければ {
   もしbuzzが0ならば {
      resultに"Buzz"を追加する
   }
   そうでなければ {
      resultにiを追加する
   }
}
```

### 4.5 繰り返し

ブロックを指定回数だけ実行します。途中で抜けるには `くり返しを抜ける` を使います。

```
N回くり返す {
   …
}
```

### 4.6 関数定義

入力（パラメータ）から答え（戻り値）を計算する手続きを定義します。

```
f = 入力 x, y に対し {
   …
   式 が答え
}
```

- パラメータなし: `入力なしに対し{ … }`
- 戻り値: `expr が答え`
- 明示的 void 戻り: `関数から抜ける`
- 関数内で定義した変数は**その関数ローカル**。外側からは見えません。
- **暗黙の戻り値**: `が答え` も `関数から抜ける` も書かずに本体が終わると、**ローカル変数をまとめたオブジェクト**が返されます。コンストラクタ風の使い方ができます。

```
point=入力x,yに対し{
   # 何も返さない → {x, y} のオブジェクトが返る
}
O=point(3,5)
>>> O["x"]
3
```

### 4.7 アサート

式の評価結果が期待値と一致することを検証する、テスト用構文です。Python の Doctest と同じ書式を採用しています。

```
>>> 式
期待値
```

### 4.8 インポート

外部の関数群を現在のプログラムから利用できるようにします。

`標準ライブラリを使う` — 標準ライブラリを現在の環境に登録。

### 4.9 何もしない

プレースホルダとして、処理を行わない文を書けます。

```
何もしない
```

### 4.10 式文（Print Expression）

代入や制御文に当てはまらない単独の式は、その評価結果が自動的に印字されます。
REPL 的な確認に便利です。

---

## 5. ブロック

複数の文をひとまとまりに扱う構文です。条件分岐・繰り返し・関数定義の本体として使われます。

- yui（明示デリミタ）: `{ … }`

---

## 6. 標準ライブラリ（主要関数）

`標準ライブラリを使う` と宣言すると、以下の関数群が呼び出せるようになります。

| 名前 | 機能 |
|------|------|
| `絶対値(x)` | 絶対値 |
| `平方根(x)` | 平方根（Float） |
| `乱数()` | 0〜1 の乱数 |
| `和`, `差`, `積`, `商`, `剰余` | 四則演算 |
| `最大値(…)`, `最小値(…)` | 最大・最小 |
| `配列化(x)`, `文字列化(x)`, `小数化(x)`, `整数化(x)` | 変換 |
| `ブール判定(x)`, `整数判定(x)`, `小数判定(x)`, `文字列判定(x)`, `配列判定(x)`, `オブジェクト判定(x)` | 型判定 |

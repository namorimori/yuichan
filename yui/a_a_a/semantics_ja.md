# Yui 言語 意味論（セミフォーマル・ビッグステップ操作的意味論）

本書は Yui 言語（`yui` 方言を基準とする抽象構文）の実行時意味を
ビッグステップ（自然意味論）形式で定義する。構文の詳細は
[`ebnf_ja.md`](ebnf_ja.md) を参照。

記法は Winskel / Pierce 流の推論規則に寄せた **セミフォーマル** な記述で、
現時点では未確定部分を `TODO` として明示する。

---

## 1. 抽象構文（要約）

`ebnf_ja.md` の具象構文を脱糖した抽象構文。

```
v ∈ Value
x ∈ Name
n ∈ ℤ        f ∈ ℝ        s ∈ String

e ::= null | true | false | n | f | s              (リテラル)
    | x                                            (変数参照)
    | [e₁,…,eₖ]                                    (配列リテラル)
    | {s₁:e₁,…,sₖ:eₖ}                              (オブジェクトリテラル)
    | e[e']                                        (添字)
    | e(e₁,…,eₖ)                                   (関数呼び出し)
    | -e                                           (単項マイナス)
    | eの大きさ                                     (長さ)

s ::= x = e                                        (代入)
    | x[e₁]…[eₖ] = e                               (インデックス代入)
    | xを増やす | xを減らす                          (増減)
    | xにeを追加する                                (末尾追加)
    | もし c ならば B₁ [そうでなければ B₂]            (分岐)
    | e 回くり返す B                                (反復)
    | x = 入力 x₁,…,xₖ に対し B                     (関数定義)
    | e が 答え                                     (値を返す)
    | 関数から抜ける                                 (無返り値 return)
    | くり返しを抜ける                               (break)
    | 何もしない                                     (pass)
    | 標準ライブラリを使う                           (import)
    | >>> e ⤶ lit                                  (doctest assertion)
    | e                                            (式文)

c ::= e₁ が e₂                                     (等値)
    | e₁ が e₂ 以外 | より小さい | より大きい | 以下 | 以上
    | e₁ が e₂ のいずれか | のいずれでもない
B ::= { s₁ … sₖ }                                  (ブロック)
```

## 2. 値・環境・ストア

### 2.1 値ドメイン

```
Value  ∋ v ::= null | true | false | n | f | s | a | o | clo
Int    ∋ n
Float  ∋ f
Str    ∋ s        (内部表現は文字コード列。§6.2)
Addr   ∋ a, o     (配列・オブジェクトは参照で同一性を持つ)
Clo    ∋ clo = ⟨λ(x₁,…,xₖ). B, ρ⟩
```

配列・オブジェクト・クロージャは **ヒープ上の参照値**。ℤ/ℝ/Bool/Null/
String は値型。

### 2.2 環境とストア

```
Env   ρ : Name ⇀ Value            (変数束縛)
Heap  H : Addr ⇀ Array ∪ Object ∪ Clo
状態  σ = (ρ, H)
```

代入・関数呼び出しは `ρ` を書き換え、配列のインデックス代入・
`追加する` は `H` を書き換える。

### 2.3 判定形式

```
式:  σ ⊢ e ⇓ v             (e を σ の下で評価すると値 v になる)
文:  σ ⊢ s ⇓ σ'            (σ を σ' に遷移させる)
条件: σ ⊢ c ⇓ b            (b ∈ {true,false})
ブロック: σ ⊢ B ⇓ σ' | Ret v | Brk
           (正常終了 / 値を返して脱出 / break)
```

`Ret v` と `Brk` は制御ジャンプを表す補助的結果で、
囲む関数／ループで捕捉される。

## 3. リテラルと変数

```
─────────────── (E-Null)       ─────────────── (E-True)
σ ⊢ 値なし ⇓ null              σ ⊢ 真 ⇓ true

─────────────── (E-False)      ─────────────── (E-Int)
σ ⊢ 偽 ⇓ false                 σ ⊢ n ⇓ n

─────────────── (E-Float)      ─────────────── (E-Str)
σ ⊢ f ⇓ f                      σ ⊢ "…" ⇓ s

        x ∈ dom(ρ)
    ────────────────── (E-Var)
    (ρ,H) ⊢ x ⇓ ρ(x)
```

未定義変数は **実行時エラー** (`undefined-variable`) を送出する
（規則を導出できない状態）。

### 3.1 文字列補間

`"…{e}…"` は抽象構文上 `補間(["…", e, "…"])` に脱糖し、
各要素を評価して文字列連結した結果を値とする。**TODO**: 正式な脱糖規則。

## 4. 複合式

### 4.1 配列・オブジェクトリテラル

```
σ₀ ⊢ e₁ ⇓ v₁   σ₁ ⊢ e₂ ⇓ v₂   …   σₖ₋₁ ⊢ eₖ ⇓ vₖ
a ∉ dom(Hₖ)       H' = Hₖ[a ↦ [v₁,…,vₖ]]
─────────────────────────────────────────────────── (E-Array)
σ₀ ⊢ [e₁,…,eₖ] ⇓ a      (状態は (ρₖ, H') に更新)
```

オブジェクトリテラルも同様に新アドレス `o` を割り当てて
`{s₁↦v₁,…}` をヒープに確保する (E-Object)。

> 注: 式評価中の副作用（関数呼び出し・配列代入）で `H` が変化するため、
> 厳密には `σ ⊢ e ⇓ v ⫶ σ'` のように状態を伝搬する判定に直す必要がある。
> **TODO**: 判定形式を `σ ⊢ e ⇓ v ⫶ σ'` に統一する。

### 4.2 添字

```
σ ⊢ e ⇓ a    σ ⊢ e' ⇓ n    H(a) = [v₀,…,vₘ₋₁]    0 ≤ n < m
──────────────────────────────────────────────────────── (E-Index-Arr)
σ ⊢ e[e'] ⇓ vₙ

σ ⊢ e ⇓ o    σ ⊢ e' ⇓ s    H(o) = {…, s↦v, …}
──────────────────────────────────────────────── (E-Index-Obj)
σ ⊢ e[e'] ⇓ v
```

範囲外・未定義キーはエラー。**TODO**: 負のインデックスの扱い。

### 4.3 単項マイナス・長さ

```
σ ⊢ e ⇓ n                      σ ⊢ e ⇓ a    H(a) = [v₀,…,vₘ₋₁]
──────────────── (E-Neg-Int)   ────────────────────────────── (E-Len-Arr)
σ ⊢ -e ⇓ -n                    σ ⊢ eの大きさ ⇓ m

σ ⊢ e ⇓ s    |s| = m
──────────────────── (E-Len-Str)
σ ⊢ eの大きさ ⇓ m
```

### 4.4 関数呼び出し

```
σ ⊢ e ⇓ clo       clo = ⟨λ(x₁,…,xₖ). B, ρ_c⟩
σ ⊢ eᵢ ⇓ vᵢ       (i = 1..k)
ρ' = ρ_c[x₁↦v₁,…,xₖ↦vₖ]
(ρ', H) ⊢ B ⇓ r
────────────────────────────────────── (E-Call)
σ ⊢ e(e₁,…,eₖ) ⇓ result(r)
```

ただし
- `r = Ret v` のとき `result(r) = v`
- `r = σ'` (正常終了) のとき `result(r) = ρ'` を走査し、関数内ローカル変数を
  オブジェクトにまとめた値を返す（`ebnf_ja.md` §8 の暗黙の戻り値）
- `r = Brk` が関数境界に届くのはエラー **TODO**

組み込み関数（`和`, `剰余`, …）は別ドメイン `Native ⊂ Value` として
`σ ⊢ native(v₁,…,vₖ) ⇓ v` の外部規則で定義する（§6）。

## 5. 条件と文

### 5.1 条件

```
σ ⊢ e₁ ⇓ v₁     σ ⊢ e₂ ⇓ v₂     v₁ = v₂
─────────────────────────────────────── (C-Eq-T)
σ ⊢ e₁ が e₂ ⇓ true

σ ⊢ e₁ ⇓ v₁     σ ⊢ e₂ ⇓ v₂     v₁ ≠ v₂
─────────────────────────────────────── (C-Eq-F)
σ ⊢ e₁ が e₂ ⇓ false
```

他の接尾辞 (`以外`/`より小さい`/`より大きい`/`以下`/`以上`/
`のいずれか`/`のいずれでもない`) も同様の二段規則で定義する。**TODO**。

等価性の詳細:
- 数値同士: `=` は数学的等値（int と float の比較は値で判定）
- 文字列同士: 文字コード列の要素ごと一致
- 配列同士: 要素ごと再帰比較（長さ一致 + 各要素等値）
- 配列 vs 文字列: 文字コード列と一致なら等値 (`yuiarray.equals` に準拠)
- オブジェクト同士: **TODO**（現状は参照等値）

### 5.2 代入・増減・追加

```
σ ⊢ e ⇓ v
───────────────────────────── (S-Assign)
(ρ,H) ⊢ x = e ⇓ (ρ[x↦v], H)

ρ(x) = n ∈ ℤ
───────────────────────────── (S-Inc)
(ρ,H) ⊢ xを増やす ⇓ (ρ[x↦n+1], H)

ρ(x) = n ∈ ℤ
───────────────────────────── (S-Dec)
(ρ,H) ⊢ xを減らす ⇓ (ρ[x↦n-1], H)

ρ(x) = a       H(a) = [v₀,…,vₘ₋₁]       σ ⊢ e ⇓ v
H' = H[a ↦ [v₀,…,vₘ₋₁, v]]
──────────────────────────────────────────────────── (S-Append)
(ρ,H) ⊢ xにeを追加する ⇓ (ρ, H')
```

インデックス代入 `x[e₁]…[eₖ] = e` も同様に `H` のみを更新する (S-SetItem, **TODO**)。

### 5.3 条件分岐

```
σ ⊢ c ⇓ true     σ ⊢ B₁ ⇓ r
─────────────────────────── (S-If-T)
σ ⊢ もし c ならば B₁ そうでなければ B₂ ⇓ r

σ ⊢ c ⇓ false    σ ⊢ B₂ ⇓ r
─────────────────────────── (S-If-F)
σ ⊢ もし c ならば B₁ そうでなければ B₂ ⇓ r
```

`そうでなければ` 省略時は C-False のとき σ をそのまま返す。

### 5.4 反復

```
σ ⊢ e ⇓ 0
──────────────────────── (S-Rep-0)
σ ⊢ e 回くり返す B ⇓ σ

σ ⊢ e ⇓ n  (n > 0)     σ ⊢ B ⇓ σ₁     σ₁ ⊢ (n-1) 回くり返す B ⇓ r
───────────────────────────────────────────────────────────── (S-Rep-N)
σ ⊢ e 回くり返す B ⇓ r

σ ⊢ e ⇓ n  (n > 0)     σ ⊢ B ⇓ Brk
────────────────────────────────── (S-Rep-Brk)
σ ⊢ e 回くり返す B ⇓ σ
```

`Ret v` がループ本体から上がってきたときは、そのまま外側に伝播させる
（規則略、**TODO**）。

### 5.5 関数定義・return・break

```
clo = ⟨λ(x₁,…,xₖ). B, ρ⟩
──────────────────────────────────────── (S-FunDef)
(ρ,H) ⊢ x = 入力 x₁,…,xₖ に対し B ⇓ (ρ[x↦clo], H)

σ ⊢ e ⇓ v
──────────────────────── (S-Return)
σ ⊢ eが答え ⇓ Ret v

────────────────────────── (S-ReturnNone)
σ ⊢ 関数から抜ける ⇓ Ret null

────────────────────────── (S-Break)
σ ⊢ くり返しを抜ける ⇓ Brk
```

### 5.6 ブロック

```
σ₀ ⊢ s₁ ⇓ σ₁   σ₁ ⊢ s₂ ⇓ σ₂   …   σₖ₋₁ ⊢ sₖ ⇓ r
───────────────────────────────────────────────── (B-Seq)
σ₀ ⊢ { s₁ … sₖ } ⇓ r
```

途中の `sᵢ` が `Ret v` / `Brk` を返したら、以降の文は評価せずその値を
ブロック全体の結果とする。

### 5.7 doctest assertion

```
σ ⊢ e ⇓ v    v = ⟦lit⟧
────────────────────────── (S-Assert-OK)
σ ⊢ >>> e ⤶ lit ⇓ σ
```

不一致は実行時アサーション失敗。**TODO**: 浮動小数点の許容誤差。

### 5.8 import

`標準ライブラリを使う` は §6 の組み込み束縛 `ρ_std` を現在の環境に
マージする。

```
───────────────────────────────────── (S-ImportStd)
(ρ,H) ⊢ 標準ライブラリを使う ⇓ (ρ ⊎ ρ_std, H)
```

## 6. 組み込み関数の意味

標準ライブラリの主要関数を外部的に定義する。**TODO**: 全関数を列挙。

### 6.1 算術

```
和(n,m)   = n + m                        (int×int → int)
和(x,y)   = x + y                        (float が絡めば float)
差(x,y)   = x - y
積(x,y)   = x * y
商(n,m)   = ⌊n / m⌋                      (int×int → int; m ≠ 0)
商(x,y)   = x / y                        (float があれば通常除算)
剰余(n,m) = n - m*⌊n/m⌋                  (int×int; m ≠ 0)
```

ゼロ除算はエラー。混合型の昇格規則: int が一方でも float なら float に
昇格する。

### 6.2 文字列・配列

文字列は内部的に文字コード列として扱う（§2.1）。このため:

- `"Hi"の大きさ = 2`
- `[72,105] = "Hi"` は true（配列と文字列の等値、§5.1）

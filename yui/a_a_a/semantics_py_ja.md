# Yui 言語 意味論（Python 対応版）

本書は Yui の実行時意味を **Python プログラムへの翻訳** として記述する。
「この Yui コードは、おおむねこの Python コードと同じ意味だ」という
対応づけを積み重ねることで、言語全体の意味を定義する。

厳密な推論規則版は [`semantics_ja.md`](semantics_ja.md)、
擬似コード版は [`semantics_semi_ja.md`](semantics_semi_ja.md) を参照。

---

## 0. なぜ Python 対応で書けるのか

Yui の参照実装は Python (`yuichan/`) で書かれており、
値表現・演算・参照セマンティクスは基本的に Python のそれを踏襲する。

- 配列 = Python の `list`
- オブジェクト = Python の `dict`
- 関数 = クロージャ（レキシカルスコープ）
- `Int` / `Float` の昇格規則 = Python と同じ
- 参照型の共有 = Python と同じ（代入でコピーしない）

ただし完全に同一ではない。主な違い:

| 項目 | Yui | Python |
|------|-----|--------|
| 文字列の内部表現 | 文字コード列 | `str` オブジェクト |
| 配列と文字列の等値 | `[72,105] == "Hi"` は真 | 偽 |
| `商` の Int×Int | 床除算 | `/` は真除算、`//` が床 |
| 算術の書き方 | `和(x,y)` 等の関数 | `+` 等の中置 |
| 変数スコープ | 関数スコープ（代入で作成） | 同左 |
| 関数の暗黙戻り値 | ローカル変数をまとめた dict | `None` |

以下、構文ごとに Python との対応を示す。

---

## 1. リテラル

| Yui | Python |
|-----|--------|
| `値なし` | `None` |
| `真` / `偽` | `True` / `False` |
| `42` | `42` |
| `3.14` | `3.14` |
| `"Hi"` | `"Hi"`（ただし内部は `[72, 105]` 相当） |
| `[1, 2, 3]` | `[1, 2, 3]` |
| `{"a": 1, "b": 2}` | `{"a": 1, "b": 2}` |

---

## 2. 変数と代入

```
x = 10
y = x
```

```python
x = 10
y = x
```

未定義変数の参照はエラー（Python の `NameError` に相当）。
変数は **関数スコープ**で管理され、代入した瞬間にスコープに追加される
（Python と同じ）。

---

## 3. 算術

Yui の算術は標準ライブラリ関数。中置記号は使えない。

| Yui | Python 相当 |
|-----|-------------|
| `和(a, b)` | `a + b` |
| `差(a, b)` | `a - b` |
| `積(a, b)` | `a * b` |
| `商(a, b)`（両 Int） | `a // b` |
| `商(a, b)`（Float 混在） | `a / b` |
| `剰余(a, b)` | `a % b` |
| `小数化(n)` | `float(n)` |
| `絶対値(x)` | `abs(x)` |

**違い**: Python の `a / b` は常に Float を返すが、Yui の `商` は
両オペランドが Int なら Int（床除算）を返す。小数の答えが欲しい場合は
片方を `小数化` で Float に昇格させる。

---

## 4. 条件分岐

```
もしxが0ならば{
   y=1
}そうでなければ{
   y=2
}
```

```python
if x == 0:
    y = 1
else:
    y = 2
```

比較接尾辞の対応:

| Yui 条件 | Python |
|----------|--------|
| `aがbならば` | `a == b` |
| `aがb以外ならば` | `a != b` |
| `aがbより小さいならば` | `a < b` |
| `aがbより大きいならば` | `a > b` |
| `aがb以下ならば` | `a <= b` |
| `aがb以上ならば` | `a >= b` |
| `aがbのいずれかならば` | `a in b` |
| `aがbのいずれでもないならば` | `a not in b` |

**等値の違い**: Yui では `[72, 105] == "Hi"` が真（文字コード列と文字列の
横断比較）。Python ではこの比較は常に偽。

---

## 5. 繰り返し

```
n回くり返す{
   ...
}
```

```python
for _ in range(n):
    ...
```

`くり返しを抜ける` は `break` に相当。Yui に `continue` は無い
（[`hamari_ja.md`](hamari_ja.md) §7 の回避策を参照）。

---

## 6. 配列と追加・増減

```
arr=[]
arrに3を追加する
arrに4を追加する
iを増やす
iを減らす
arr[0]=99
```

```python
arr = []
arr.append(3)
arr.append(4)
i = i + 1
i = i - 1
arr[0] = 99
```

配列は参照型（Python の `list` と同じ）。代入・引数渡しでコピーされない。

**違い**: Yui には `pop` / `shift` に相当する削除プリミティブがない。

---

## 7. オブジェクト（辞書）

```
person={"名前":"ゆい","年齢":10}
name=person["名前"]
person["年齢"]=11
```

```python
person = {"名前": "ゆい", "年齢": 10}
name = person["名前"]
person["年齢"] = 11
```

---

## 8. 関数定義と呼び出し

```
add=入力a,bに対し{
   和(a,b)が答え
}

x=add(3,4)
```

```python
def add(a, b):
    return a + b

x = add(3, 4)
```

クロージャはレキシカルスコープで Python と同じ。

**暗黙の戻り値**: Yui では `が答え` や `関数から抜ける` なしに
関数本体が終わると、**関数内で新規に束縛されたローカル変数を dict に
まとめた値** が返る。Python には対応機能がないが、概念的には:

```python
def f(x):
    a = x + 1
    b = x * 2
    # return が無い
```

は Yui では次に相当:

```python
def f(x):
    a = x + 1
    b = x * 2
    return {"a": a, "b": b}
```

`関数から抜ける` は `return None` に相当。

---

## 9. 文字列補間

```
name="ゆい"
greet="こんにちは、{name}さん"
```

```python
name = "ゆい"
greet = f"こんにちは、{name}さん"
```

## 10. 長さ演算

```
n=arrの大きさ
m="Hi"の大きさ
```

```python
n = len(arr)
m = len("Hi")    # ただし Yui では内部表現が文字コード列なので len=2 は同じ
```

---

## 11. doctest

```
f=入力xに対し{ 積(x,x)が答え }
>>> f(3)
9
```

```python
def f(x):
    return x * x

assert f(3) == 9
```

## 12. import

```
標準ライブラリを使う
```

```python
from yuichan.stdlib import *   # のようなイメージ
```

`和`, `剰余`, `最大値`, … が現在のスコープに追加される。

---

## 13. 式文

```
f(x)
```

```python
result = f(x)
print(result)    # REPL モードなら自動表示
```

Yui では単独の式文は評価結果を自動印字する（Python の REPL と同じ）。

## 14. まとめ: Yui プログラムの「Python 翻訳」アルゴリズム

1. `標準ライブラリを使う` を `from yuichan_stdlib import *` に
2. 関数定義 `x = 入力 a,b に対し { ... }` を `def x(a,b): ...` に
3. `もし c ならば B₁ そうでなければ B₂` を `if/else` に
4. `n 回くり返す B` を `for _ in range(n):` に
5. `xを増やす` を `x = x + 1` に、`xを減らす` を `x = x - 1` に
6. `xにeを追加する` を `x.append(e)` に
7. 算術関数呼び出し (`和`, `積`, …) を `+`, `*`, … に
8. `eが答え` を `return e` に
9. `くり返しを抜ける` を `break` に
10. `>>> e` + 次行 `lit` を `assert e == lit` に
11. 関数本体が return 無しに終わる場合、ローカル変数を集めた dict を
    末尾で `return` する
12. 文字列 `"…{e}…"` を f-string に

ただし以下は Python で厳密に再現できない:

- **文字列 = 文字コード列**: `"Hi" == [72, 105]` は Python では偽だが
  Yui では真
- **`商` の型依存挙動**: Int×Int だけ床除算になる

これらは実行時の値判定として別途扱う必要がある。詳細な規則は
`semantics_ja.md` §6 を参照。

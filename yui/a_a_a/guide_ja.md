# Yui プログラミングガイド（日本語）

Yui は日本語で書ける教育用プログラミング言語です。
小中学生でも読み書きできるように、記号を減らし、自然な日本語の語順でプログラムを組み立てます。

---

## 1. はじめてのプログラム

文字列をそのまま書くと、その値が表示されます。

```yui
# "Hello, world!" と表示する
"Hello, world!"
```

行頭から `#`（または `＃`）以降は**コメント**で、行末まで無視されます。

---

## 2. 変数と増減

`変数名 = 式` で値を結びつけます。同名に再代入すると上書きされます。
数値変数には「増やす／減らす」という簡略構文があります。

```yui
# 変数 x と y を定義する
x = 1
y = -2

# x を 1 増やす
xを増やす

# y を 1 減らす
yを減らす

>>> x
2
>>> y
-3
```

`>>> 式` と、その次の行に**期待値**を書くと、テスト（アサート）になります。
Python の doctest と同じ書き方で、Yui のプログラム内に検証を埋め込めます。

---

## 3. 型とリテラル

Yui の値は以下の 7 種類のいずれかです。

| 型 | リテラル |
|----|----------|
| ヌル | `値なし` |
| 論理値 | `真` / `偽` |
| 整数 | `42` |
| 小数 | `3.14`（表示は小数点以下 6 桁固定） |
| 文字列 | `"あ"`, `"x={x}"` |
| 配列 | `[1, 2, 3]` |
| オブジェクト | `{"x": 1, "y": 2}` |

文字列は内部的に**文字コードの配列**として保持されるため、配列と同じ操作（要素追加・長さ取得など）が使えます。

---

## 4. 文字列補間

`"{式}"` と書くと、文字列の中に式の値を埋め込めます。

```yui
name = "ゆい"
age  = 12
msg  = "こんにちは、{name}さん！あなたは{age}歳です。"
msg

>>> msg
"こんにちは、ゆいさん！あなたは12歳です。"
```

`{` や `"` そのものを書きたいときは `\{` `\"` のようにエスケープします。
`\\`, `\n`, `\t` も使えます。

---

## 5. 配列

`[...]` で配列を作り、`配列に値を追加する` で末尾に要素を足します。

```yui
A = [1, 2, 3]

# 末尾に 0 を追加する
Aに0を追加する

# 最初の要素を 1 増やす
A[0]を増やす

# 2 が含まれていれば A[0] を A[3] で上書きする
もし2がAのいずれかならば{
   A[0] = A[3]
}

>>> Aの大きさ
4
```

### 5.1 インデックスと長さ

- `a[i]` — 配列 a の i 番目の要素（0 から数える）
- `aの大きさ` — 要素数

```yui
標準ライブラリを使う
A = [10, 20, 30]
n = Aの大きさ

first = A[0]
last  = A[差(n,1)]

>>> first
10
>>> last
30

# 要素の書き換え
A[1] = 200
>>> A[1]
200
```

---

## 6. オブジェクト

`{"キー": 値, ...}` でオブジェクトを作ります。キーは文字列です。

```yui
O = {"x": 0, "y": 0}
O["x"] = 1
O["y"] = 2

>>> O["x"]
1
>>> O["y"]
2
```

---

## 7. 文字列は配列

文字列は文字コード配列なので、`[i]` で取り出したり、末尾に追加したりできます。

```yui
s = "hello"

# 最初の文字を 'H' に書き換える
s[0] = "H"[0]

# " world" の各文字を末尾に追加する
t = " world"
i = 0
tの大きさ回くり返す{
   sにt[i]を追加する
   iを増やす
}

>>> s
"Hello world"
```

---

## 7.5 整数はビット配列

整数も内部的には **2 進数のビット配列（LSB が先頭）** として保持されます。
そのため、整数に対してもインデックス・長さ取得・要素追加などの配列操作が使え、
記号を使わずにビット演算を書けます。

```yui
# 6 は二進で 110 → [0, 1, 1]（LSB 先頭）
>>> 6[0]
0
>>> 6[1]
1
>>> 6[2]
1
>>> 6の大きさ
3
```

ビット配列 ⇔ 整数の相互比較もできます。

```yui
>>> [0,1,1]が6
真
```

### 例: ビット論理積（AND）

各ビットを取り出して、両方が 1 のときだけ 1 を追加します。

```yui
bits_and = 入力A,Bに対し{
   n = Aの大きさ
   もしBの大きさがnより小さいならば{
      n = Bの大きさ
   }
   i = 0
   X = 0
   n回くり返す{
      x = 0
      もしA[i]が1ならば{
         もしB[i]が1ならば{
            x = 1
         }
      }
      Xにxを追加する
      iを増やす
   }
   Xが答え
}

>>> bits_and(6, 5)
4
>>> bits_and(3, 1)
1
```

### 例: 右シフト

先頭（LSB 側）の k ビットを捨てれば右シフトになります。

```yui
bits_rshift = 入力bits,kに対し{
   X = 0
   i = k
   bitsの大きさ回くり返す{
      もしiがbitsの大きさ以上ならば{
         くり返しを抜ける
      }
      Xにbits[i]を追加する
      iを増やす
   }
   Xが答え
}

>>> bits_rshift(6, 1)
3
>>> bits_rshift(12, 2)
3
```

同じ要領で OR・XOR・左シフト・加算・2 の補数・減算も配列操作として実装できます。

## 8. 条件分岐

`もし <式1> が <式2> <接尾辞> ならば { ... }` の形です。
`そうでなければ { ... }` は省略できます。

```yui
flag   = 真
result = 0

もしflagが真ならば{
   result = 1
}
そうでなければ{
   result = 2
}

>>> result
1
```

### 8.1 比較の接尾辞

Yui では `==` `!=` `<` `>` などの**記号は使いません**。代わりに接尾辞で比較します。

| 接尾辞 | 意味 | 例 |
|--------|------|----|
| （なし） | == | `もしxが0ならば` |
| `以外` | != | `もしxが0以外ならば` |
| `より小さい` | < | `もしrがbより小さいならば` |
| `より大きい` | > | `もしxが10より大きいならば` |
| `以下` | <= | `もしdistが1以下ならば` |
| `以上` | >= | `もしnが100以上ならば` |
| `のいずれか` | in | `もし"banana"がfruitsのいずれかならば` |
| `のいずれでもない` | not in | `もし"grape"がfruitsのいずれでもないならば` |

```yui
fruits = ["apple", "banana", "cherry"]
found   = 0
missing = 0

もし"banana"がfruitsのいずれかならば{
   foundを増やす
}
もし"grape"がfruitsのいずれでもないならば{
   missingを増やす
}

>>> found
1
>>> missing
1
```

### 8.2 多段分岐

`elif` はありません。`そうでなければ` の中に `もし` をネストします。

## 9. 繰り返し

`N回くり返す { ... }` でブロックを N 回実行します。途中で抜けるには `くり返しを抜ける` を使います。

```yui
count = 0
10回くり返す{
   countを増やす
   もしcountが5ならば{
      くり返しを抜ける
   }
}

>>> count
5
```

## 10. 関数

### 10.1 関数定義と戻り値

`入力 a, b, ... に対し { ... 式 が答え }` の形で関数を作ります。
`式 が答え` が**戻り値**、`関数から抜ける` は明示的な void 戻りです。

```yui
succ = 入力nに対し{
   nを増やす
   nが答え
}

>>> succ(0)
1
```

関数内で作った変数は**その関数ローカル**で、外からは見えません。

### 10.2 暗黙の戻り値（コンストラクタ風）

`が答え` も `関数から抜ける` も書かずに本体が終わると、
関数内で定義したローカル変数を**ひとつのオブジェクトにまとめて**返します。

```yui
point = 入力x,yに対し{
   # 何も返さない → {"x": ..., "y": ...} が返る
}
O = point(3, 5)

>>> O["x"]
3
```

### 10.3 再帰

関数は自分自身を呼び出せます。

```yui
fact = 入力nに対し{
   もしnが0ならば{
      1が答え
   }
   そうでなければ{
      積(n, fact(差(n,1)))が答え
   }
}

>>> fact(0)
1
>>> fact(5)
120
```

（`積` `差` は標準ライブラリの関数。次章で扱います。）

## 11. 標準ライブラリ

Yui では `+ - * / %` などの**算術記号は使いません**。
代わりに、ソース先頭で `標準ライブラリを使う` と宣言し、関数として呼び出します。

### 11.1 四則演算

| 関数 | 意味 |
|------|------|
| `和(x, ...)` | 合計（引数が配列なら展開） |
| `差(x, y, ...)` | 差 |
| `積(x, y, ...)` | 積 |
| `商(x, y)` | 商（int/int は床除算） |
| `剰余(x, y)` | 剰余 |

```yui
標準ライブラリを使う

gcd = 入力a,bに対し{
   a回くり返す{
      もしbが0ならば{
         くり返しを抜ける
      }
      r = 剰余(a, b)
      a = b
      b = r
   }
   aが答え
}

>>> gcd(12, 18)
6
>>> gcd(100, 75)
25
```

### 11.2 数学関数

| 関数 | 機能 |
|------|------|
| `絶対値(x)` | 絶対値 |
| `平方根(x)` | 平方根（常に小数） |
| `最大値(...)` / `最小値(...)` | 最大・最小（可変長／配列可） |
| `乱数()` | 0〜1 の小数乱数 |

```yui
標準ライブラリを使う

>>> 絶対値(-7)
7
>>> 平方根(9)
3.000000
>>> 最大値(3, 1, 4, 1, 5)
5
>>> 最小値([10, 20, 30])
10
```

### 11.3 型変換

| 関数 | 機能 |
|------|------|
| `整数化(x)` | 整数へ |
| `小数化(x)` | 小数へ |
| `文字列化(x)` | 文字列へ |
| `配列化(x)` | 文字コード配列へ |

```yui
標準ライブラリを使う

>>> 整数化("42")
42
>>> 整数化(3.700000)
3
>>> 小数化(5)
5.000000
>>> 文字列化(42)
"42"
>>> 配列化("Hi")
[72,105]
```

### 11.4 型判定

| 関数 | 機能 |
|------|------|
| `ブール判定(x)` | 論理値なら真 |
| `整数判定(x)` | 整数なら真 |
| `小数判定(x)` | 小数なら真 |
| `文字列判定(x)` | 文字列なら真 |
| `配列判定(x)` | 配列なら真 |
| `オブジェクト判定(x)` | オブジェクトなら真 |

```yui
標準ライブラリを使う

>>> 整数判定(42)
真
>>> 文字列判定("hello")
真
>>> 整数判定("42")
偽
```

## 12. 総合例: FizzBuzz

ここまでの要素を組み合わせた例です。
`和 差 積 商 剰余` を使わず、カウンタでも Fizz/Buzz を判定できることに注目してください。

```yui
result = []
i    = 0
fizz = 0
buzz = 0

100回くり返す{
   iを増やす
   fizzを増やす
   buzzを増やす
   もしfizzが3ならば{ fizz = 0 }
   もしbuzzが5ならば{ buzz = 0 }

   もしfizzが0ならば{
      もしbuzzが0ならば{
         resultに"FizzBuzz"を追加する
      }そうでなければ{
         resultに"Fizz"を追加する
      }
   }そうでなければ{
      もしbuzzが0ならば{
         resultに"Buzz"を追加する
      }そうでなければ{
         resultにiを追加する
      }
   }
}

>>> resultの大きさ
100
>>> result[2]
"Fizz"
>>> result[4]
"Buzz"
>>> result[14]
"FizzBuzz"
```

## 13. 総合例: モンテカルロ法で π を推定

関数・繰り返し・乱数・算術ライブラリを組み合わせた例です。

```yui
標準ライブラリを使う

monte_carlo = 入力nに対し{
   hits = 0
   n回くり返す{
      x = 乱数()
      y = 乱数()
      dist = 平方根(和(積(x,x), 積(y,y)))
      もしdistが1以下ならば{
         hitsを増やす
      }
   }
   商(積(小数化(hits), 4), 小数化(n))が答え
}

monte_carlo(1000)   # サンプル数が多いほど 3.14159... に近づく
```

## 14. まとめ

- **記号を減らし、日本語の語順で書く**のが Yui の特徴。
- 比較は `が〜ならば`、四則演算は `和/差/積/商/剰余`。
- 文字列は文字コード配列で、配列と統一的に扱える。
- 関数は `入力...に対し{ ...が答え }`、暗黙の戻り値はローカル変数のオブジェクト。
- `>>> 式` と期待値を書くと、プログラム内にテストを残せる。

より詳しい仕様は `pro159/spec_ja.md`、実例は `yui_examples/` を参照してください。
